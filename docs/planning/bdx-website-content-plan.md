# BDX サイト コンテンツ企画書

> 2026-03-12 作成 | 朝レビュー用

---

## 1. 背景と目的

### 現状の課題
- 新卒採用ページ（`/recruitment-graduate/`）は**募集要項のみ**で情報量が少ない
- 他コンサルファーム（アクセンチュア、NRI、ABeam 等）は充実した採用コンテンツを展開
- ジョブインターンが学生から高く評価されているが、**サイト上でアピールできていない**
- アワード受賞実績の記載もなし（T-173）
- 面接アンケートで 81% が「第一志望 or 第一志望群」→ **体験の質は高い。伝え方の問題**

### 目的
1. **ジョブインターン・面接体験の魅力を可視化**して、エントリー率向上
2. **学生の声（アンケート）を活用**して、リアルな訴求力を高める
3. **アワード受賞実績**で企業の信頼性を強化
4. 「成長×挑戦×実行」のメッセージを一貫して伝える

## 3. ページ別コンテンツ設計

### 3-1. ジョブインターン紹介ページ（P0）

**目的**: 「BDX のインターンはこんなに面白い」を伝え、エントリーの動機付けを強化

**セクション構成**:

1. **ヒーローセクション**
   - キャッチコピー例: 「実際の経営課題に挑む。それが B&DX のジョブインターン。」
   - ビジュアル: インターン風景の写真（要手配）

2. **インターンの特徴**（3つの柱）
   - **リアルなケース**: 架空ではない実際の業界課題に取り組む
   - **少人数精鋭**: 現役コンサルタントと密に議論できる環境
   - **フィードバック重視**: 1人1人に対する丁寧なフィードバック

3. **プログラム概要**
   - 期間・スケジュール
   - テーマ例（過去の実績から抽象化）
   - 参加条件

4. **参加者の声**（アンケートから抽出）
   > 「面接官の方に自分の論点の至らない点を指摘していただいたり、論理の飛躍を訂正できるよう誘導していただいたことで、非常に多くの学びが得られた」
   > 「単に戦略を描くのではなく、実装まで見据えて思考する姿勢に触れ、自身が体感した"実行の重要性"と重なり、より強く惹かれました」

5. **エントリーCTA**
   - Sonar エントリーフォームへのリンク

### 3-2. 学生の声ページ（P0）

**目的**: 面接後アンケートのリアルな声で「ここで働きたい」を醸成

**セクション構成**:

1. **数字で見る B&DX の面接体験**
   - 志望度: 81% が第一志望 or 第一志望群
   - 理解度: 96% が「理解できた」以上
   - 就活軸: やりがい 32%、成長スピード 21%、仕事内容 21%

2. **テーマ別の声**（カード UI）

   **🔷 「戦略×実行の一気通貫」が最大の差別化要因**（最多テーマ）
   > 「面接を通じて『戦略を描くだけでなく、現場で実行までやり切る組織』であるという印象に変わりました」
   > 「構想だけでなく現場に落ちる形まで設計し、定着まで伴走して成果に繋げる実行力に強みがある会社」

   **🔷 「プロフェッショナルだが温かい」**
   > 「面接官の皆様が非常に物腰柔らかく、私の話に真摯に耳を傾けてくださる温かさに触れました」
   > 「『何を伝えようとしているのか』を真摯に汲み取ろうとしてくださる姿勢に温かさを感じた」

   **🔷 「成長環境・若手の裁量」**
   > 「想像以上に育成を大切にされている会社だと感じました」
   > 「ベンチャーファームでありながらクライアントの規模がかなり大きいことがわかり、大きな裁量と責任を持ちながら日本社会に貢献できる環境に魅力を感じました」

   **🔷 「ケース面接で思考力重視を実感」**
   > 「論点整理の鋭さだけでなく、『実際にどう動かすか』『現場で実行可能か』という観点を重視されていると感じました」

## 4. 訴求メッセージの一貫性

アンケート分析から導かれた **BDX の4つの強み**をサイト全体で一貫して訴求:

| # | 強み | 裏付けデータ | 訴求箇所 |
|---|------|-------------|----------|
| 1 | **戦略×実行の一気通貫** | アンケート47%が言及 | 全ページ、特にインターン・声 |
| 2 | **プロフェッショナル×温かさ** | アンケート32%が言及 | 声ページ、社員インタビュー |
| 3 | **成長環境・若手の裁量** | アンケート26%が言及 | キャリアパス、インターン |
| 4 | **思考力重視のケース面接** | アンケート21%が言及 | インターン、選考フロー |

## 5. 実装アプローチ

### 技術方針
- 既存サイトの HTML/CSS 構造をそのまま踏襲
- 新規ページは既存ページのテンプレート（ヘッダー・フッター・CSS）を流用
- レスポンシブ対応は既存 CSS の仕組みに従う
- 画像は `/images/recruitment/` 以下に配置

### 制作フロー
1. **コンテンツ原稿作成** → Owner レビュー
2. **HTML 実装** → GitHub push → ステージング（GitHub Pages）で確認
3. **Owner + 安部さん承認**
4. **本番 FTP デプロイ**（GitHub Actions workflow_dispatch）

### 優先順位
1. 🔴 P0: ジョブインターン紹介 + 学生の声（同時着手）
2. 🟠 P1: アワード受賞実績（受賞名の確認が取れ次第）
3. 🟡 P2: 社内イベント + キャリアパス（P0完了後）
